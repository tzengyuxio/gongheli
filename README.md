# 共和曆 (Gonghe Calendar)

> 聖人慎守日月之數，以察星辰之行，以序四時之順逆，謂之厤 ——《大戴禮記·曾子天圓》

**簡以致用，精以合天**

共和曆是一套以西周共和元年為紀元的純太陽曆。

## 曆法規則

### 紀元

以**西周共和元年**（西元前 841 年）為曆法紀元——共和 1 年起始於 842 BC 冬至，大致涵蓋歷史上的共和元年。曆法採零年設計，共和 0 年存在（約西元前 843 年），消除了公元紀年無零年的問題。

### 歲首

以冬至對齊為歲首（**算術歲首**）：創曆時以天文冬至定位 1 月 1 日，之後嚴格依閏法推算，不再每年重新觀測。長期而言，1 月 1 日與天文冬至之間會有輕微漂移，但仍在合理範圍內。

### 月制

| 月份 | 天數 |
|------|------|
| 單數月（1, 3, 5, 7, 9, 11） | 30 天 |
| 偶數月（2, 4, 6, 8, 10） | 31 天 |
| 12 月 | 平年 30 天，閏年 31 天 |

閏日置於**年末**（12 月），因此每年 1～11 月的日期與星期的相對位置固定不變（不受閏日影響），這是共和曆相較於格里曆（閏日在 2 月，導致 3 月後星期位移）的優點之一。

### 閏法

```
Y % 4 == 0  AND  Y % 128 != 0
```

四年一閏，128 的倍數年不閏。共和 0 年亦不閏（0 % 128 == 0）。

每 128 年含 31 閏，平均年長 46751/128 ≈ 365.242188 天，約 400,000 年才累積一天的偏差，精確度為格里曆（97/400 規則）的 150 倍。

### 紀

**4418 年**為一紀，此為回歸年、朔望月、干支甲子（60 日）、七曜週（7 日）的近似最小公倍數週期：

| 單位 | 數量 |
|------|------|
| 年 | 4,418 |
| 朔望月 | 54,643 |
| 日 | 1,613,640 |
| 甲子 | 26,894 |
| 週 | 230,520 |

### 曜日與時間

- **曜日**：沿用現行七曜名稱（週一～週日），不另創新詞
- **週首**：週一為一週之始。共和 1 年 1 月 1 日 = 週一，週序由此自然延伸
- **日內時間**：沿用 24 小時制，以午夜（00:00）為日的起點

### 子輿日

連續紀日系統，紀念**曾子**（名參，字子輿），以午夜為日的起始（有別於儒略日以正午起算）。

| 對應關係 | 數值 |
|---------|------|
| 子輿日 0 對應儒略日 | 613,270.5 |
| 共和 0 年 1 月 1 日 | 子輿日 800,609 |
| 第 0 紀首日（子輿日起點） | 子輿日 0 |
| 第 1 紀首日（西元 1384 年 12 月 13 日） | 子輿日 1,613,640 |

## 設計理念

共和曆的目標是一套**跨越空間與時間的通用曆法**——不偏向任何單一文化或歷史起點，為所有人建立一套公平、精確的時間座標系。此理念近似世界語（Esperanto）的精神：以人工設計消弭隔閡。本專案前身 `kalendaro` 正是取自世界語中「曆法」一詞。

## 工具

`tools/` 目錄收錄研究輔助工具，均可直接執行：

| 工具 | 用途 |
|------|------|
| `find_cycle.py` | 搜尋回歸年、朔望月、甲子、七曜的近似最小公倍週期 |
| `leap_year.py` | 搜尋最佳閏年規則 |
| `fraction.py` | 浮點數最佳有理數近似，推導閏法分數 |
| `solstice_new_moon.py` | 搜尋朔旦冬至甲子日，確認第一紀起始錨點 |

### 環境設置

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

`solstice_new_moon.py` 需要星曆資料（首次使用時下載，約 623 MB）：

```bash
python tools/solstice_new_moon.py --download
python tools/solstice_new_moon.py --start -900 --end 1400
```
