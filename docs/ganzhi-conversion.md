# 干支紀日與共和曆、子輿日的換算

> 草稿備案，供日後撰文使用。
> 分析日期：2026-03-06

## 核心發現

**子輿日 0 就是甲子日**，因此干支換算公式極為簡潔：

```
ganzhi_index = ZDN % 60
```

其中 0 = 甲子、1 = 乙丑、……、59 = 癸亥。

這不是巧合，而是設計的必然結果：子輿日的原點由第 1 紀錨點（朔旦冬至**甲子**日）反推而來，而一紀恰好是 60 的整數倍（1,613,640 = 26,894 × 60），所以往前推一紀必然仍落在甲子日。

## 換算公式一覽

### 1. ZDN → 干支

```
ganzhi_index = ZDN % 60
```

最簡形式。ZDN 0 = 甲子日。

### 2. JDN → 干支

```
ganzhi_index = (JDN − 11) % 60
```

JDN 11 為甲子日（傳統干支紀日的參考錨點）。

### 3. ZDN ↔ JDN

```
ZDN = JDN − 613,271
JDN = ZDN + 613,271
```

其中 613,271 = JDN\_ZD0，子輿日 0 對應的儒略日數。

### 4. 共和曆 → ZDN

```
ZDN = (Y−1)×365 + ⌊(Y−1)/4⌋ − ⌊(Y−1)/128⌋ + cum_days[M−1] + D + 800,973
```

其中：
- `cum_days` = [0, 30, 61, 91, 122, 152, 183, 213, 244, 274, 305, 335]（各月累積天數）
- 800,973 = ZD\_EPOCH = ZD\_GH\_Y1\_M1\_D1 − 1

### 5. 共和曆 → 干支（合併公式）

```
day_count = (Y−1)×365 + ⌊(Y−1)/4⌋ − ⌊(Y−1)/128⌋ + cum_days[M−1] + D
ganzhi_index = (day_count + 33) % 60
```

其中 33 = 800,973 mod 60 = ZD\_EPOCH mod 60。

## 週期整除性

| 週期 | 天數 | mod 60 | mod 7 | 意義 |
|------|------|--------|-------|------|
| 一紀（4,418 年） | 1,613,640 | **0** | **0** | 干支、星期同時歸零 |
| 128 年閏法週期 | 46,751 | 11 | — | 每個週期干支推進 11 天 |
| 4 年小週期 | 1,461 | 21 | — | 每 4 年干支推進 21 天 |

一紀能同時整除 60（干支）和 7（星期），這正是 4,418 年被選為紀的長度的原因——它是回歸年、朔望月、干支甲子、七曜週的近似最小公倍數。

## 附註：第 1 紀錨點的儒略曆/格里曆差異

第 1 紀錨點（朔旦冬至甲子日）在歷史文獻中記為**明洪武十七年**，當時使用的是儒略曆（大統曆），對應：

- 儒略曆：1384 年 12 月 13 日
- 格里曆（proleptic）：1384 年 12 月 21 日
- 差距：8 天（14 世紀的儒略曆—格里曆偏移量）

程式碼中使用 proleptic Gregorian，因此以 `gregorian_to_jdn(1384, 12, 21)` 得到正確的 JDN 2,226,911。網站與文件中提到「1384 年 12 月 13 日」時，指的是儒略曆日期。

## 設計上的優雅之處

1. **ZDN % 60 = 干支**：無需任何偏移量，最簡可能的公式
2. **一紀整除 60 和 7**：每經過一紀，干支與星期同時回到原點
3. **子輿日原點 = 甲子日**：由第 1 紀錨點（天文實測的朔旦冬至甲子日）反推而來，天文與算術完美對齊
